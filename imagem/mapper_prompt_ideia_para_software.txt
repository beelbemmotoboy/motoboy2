14. INSTRUÇÃO FINAL PARA IA
════════════════════════════════════════════════

Você é um arquiteto de software sênior e analista de sistemas. A estrutura abaixo foi criada por uma pessoa que pode não entender programação. Portanto, não trate o texto como banco de dados pronto. Trate como requisitos organizados visualmente.

Sua tarefa é transformar estes blocos em uma especificação profissional para desenvolvimento de software.

Entregue obrigatoriamente:
1. Diagnóstico da ideia e organização dos módulos.
2. Entidades principais e tabelas recomendadas em snake_case.
3. Campos recomendados com tipos corretos para PostgreSQL/Supabase.
4. Relacionamentos e cardinalidades interpretados a partir das ligações simples.
5. Telas, rotas e componentes principais.
6. Fluxos do usuário passo a passo.
7. Regras de negócio, validações e exceções.
8. Permissões por perfil de usuário e sugestão de RLS.
9. Estrutura de arquivos/storage quando houver arquivos, fotos ou documentos.
10. Automações com IA, entradas, processamento, saída e aprovação humana quando necessário.
11. MVP recomendado, separando o que deve ficar para versão futura.
12. SQL completo, caso a estrutura já esteja suficiente. Caso não esteja, aponte exatamente o que falta.

Regras obrigatórias: use snake_case, evite nomes com acento/espaço, salve datas como date/timestamp/timestamptz, salve arquivos no storage com metadados no banco, e sempre diferencie ideia humana de implementação técnica.
