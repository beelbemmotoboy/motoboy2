Crie um sistema web app responsivo, mobile-first, chamado Beelbem Motoboy.

O Beelbem Motoboy será um sistema para organizar entregas locais entre lojistas e motoboys. O lojista poderá cadastrar uma entrega manualmente ou tirar foto da comanda/pedido. O sistema deverá salvar a foto, preparar a estrutura para análise por IA/OCR via n8n, preencher os dados da entrega e procurar um motoboy disponível para aceitar a corrida.

O sistema deve funcionar como um aplicativo mobile rodando no navegador, com foco em simplicidade, estabilidade e uso real por lojistas, motoboys, administradores de cidade e administrador geral do sistema.

IMPORTANTE SOBRE O MVP:
Este é um MVP funcional, mas com estrutura preparada para crescimento.

Neste momento:
- Não criar cálculo de rota avançado.
- Não criar navegação curva a curva.
- Não criar otimização complexa estilo Uber.
- Não criar marketplace para consumidor final.
- Não criar chat interno.
- Não criar pagamento online automático completo.
- Não criar integração obrigatória com WhatsApp.
- Não criar IA/OCR real obrigatória dentro do Lovable.

O mapa será usado para:
- Acompanhamento visual das entregas.
- Mostrar loja, motoboy e destino quando houver coordenadas.
- Atualizar localização do motoboy durante a entrega.
- Ajudar o lojista e o admin a saberem onde a entrega está.

O sistema deve deixar a estrutura pronta para futuramente integrar:
- n8n
- IA/OCR para leitura da comanda
- WhatsApp
- cálculo automático de rota
- cálculo automático de frete
- pagamento Pix automatizado

9. LÓGICA PARA ENCONTRAR MOTOBOY
====================================================================

Criar função:
procurarMotoboyDisponivel

Objetivo:
Escolher um motoboy adequado para receber a entrega.

Regra inicial do MVP:
- Buscar motoboys com status_motoboy = "disponivel".
- Filtrar pela mesma cidade da loja.
- Priorizar motoboys com localização recente.
- Priorizar motoboy mais próximo da loja quando houver coordenadas.
- Considerar rank/estrelas do motoboy, quando existir.
- Se não houver coordenadas, usar cidade e bairro como referência textual.
- Criar solicitação para o motoboy escolhido.
- Alterar status da entrega para "aguardando_aceite".

Critérios de escolha, em ordem:
1. Motoboy disponível na mesma cidade.
2. Motoboy com localização mais recente.
3. Motoboy mais próximo da loja, se houver latitude e longitude.
4. Melhor avaliação/rank, se houver.
5. Motoboy com menor quantidade de entregas em andamento.
6. Primeiro disponível, se não houver dados suficientes.

Se o motoboy estiver em movimento para outra entrega:
- Neste MVP, não criar roteirização complexa.
- Apenas permitir que o sistema considere o motoboy como disponível se o status dele estiver "disponivel".
- Futuramente, poderá existir status "disponivel_em_rota".

Se nenhum motoboy estiver disponível:
- Manter entrega com status "aguardando_motoboy".
- Mostrar ao lojista:
  "Nenhum motoboy disponível no momento. A entrega ficará na fila."

Quando encontrar motoboy:
- Criar registro em solicitacoes_de_entrega.
- Mostrar ao lojista:
  "Entrega enviada para um motoboy disponível."
- Quando o motoboy aceitar, mostrar:
  "Motoboy a caminho."

Salvar também:
- Horário em que a entrega foi criada.
- Horário em que a solicitação foi enviada.
- Horário em que o motoboy aceitou.
- Tempo até aceite.

11. FLUXO DA ENTREGA
====================================================================

Status da entrega:

1. aguardando_motoboy
Entrega criada, mas ainda sem motoboy.

2. aguardando_aceite
Entrega enviada para um motoboy, aguardando resposta.

3. em_andamento
Motoboy aceitou e está realizando a entrega.

4. entregue
Motoboy marcou como entregue.

5. concluida
Entrega finalizada e validada no sistema.

6. cancelada
Entrega cancelada pelo lojista ou admin.

Fluxo:
1. Lojista cria entrega.
2. Sistema salva com status aguardando_motoboy.
3. Sistema procura motoboy.
4. Sistema envia solicitação.
5. Entrega muda para aguardando_aceite.
6. Motoboy aceita.
7. Entrega muda para em_andamento.
8. Motoboy vai até a loja e depois até o cliente.
9. Motoboy clica em "Entregue".
10. Sistema salva coordenada do motoboy no momento da entrega.
11. Sistema vincula essa coordenada ao pedido/cliente.
12. Entrega muda para entregue ou concluida.
13. Sistema calcula valor a pagar ao motoboy.
14. Sistema atualiza financeiro do lojista e do motoboy.

- Pode gerenciar cidades.
